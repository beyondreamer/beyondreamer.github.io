一、面向对象和面向过程的区别

C 完全面向过程
C++ 一半面向过程一半面向对象
Java 完全面向对象

1.什么是面向过程的开发方式

  1. 特点
    ①注重步骤
    ②注重因果关系
  • 缺点
    代码和代码之间耦合度太高，扩展力太差
  • 优点
    小型项目，采用面向过程方式进行开发，效率高

    2.什么是面向对象的开发方式

  1. 将现实世界分隔成几个小的单元，然后每一个单元都实现成对象，然后驱动一下，让各个对象之间协作起来形成一个系统
  2. 特点
    耦合度低，扩展力强

    3.采用面向对象的方式贯穿整个系统，涉及三个术语

    OOA：面向对象分析
    OOD：面向对象设计
    OOP：面向对象编程

分析->设计->编程

4.面向对象包括三大特征

  1. 封装
  2. 继承
  3. 多态

    5.类和对象的概念

  4. 什么是类
    一种模板，描述所有对象的共同特征信息
  5. 什么是对象
    实际存在的个体

Java语言中，要想得到对象，必须先定义一个类，对象是通过类这个模板创造出来的

术语

类 对象 实例 实例化 抽象

二、类的定义

1．定义

[修饰符列表] class 类名{} 类体=属性加方法

属性在代码上以变量的形式存在
属性以数据的形式存在（成员变量）

public class xueSheng{
    int xueHao;
}

java中所有的类都属于引用数据类型

2．对象的创建和使用

  1. new 类名 ()

  2. new是个运算符专门负责对象的创建

     xueSheng s = new xueSheng ();

关于编译的过程
3. 对于编译器来说先编译test.java会自动找xuesheng.class，如果没有会自动编译xuesheng.java生成.class 文件

3．成员变量

没有手动赋值，系统默认赋值

byte 0                short 0     
int 0                  long 0L  
float 0.0F          double 0.0  
boolean false      char \u0000 引用数据类型 null

4．实例变量

  1. 对象级别的变量
  2. 不能通过类名来直接访问实例变量

对象和引用的区别

  1. 对象是通过new出来的，在堆内存中储存
  2. 引用但凡是变量，并且该变量中保存了内存地址指向了堆内存中的对象
  3. 怎么访问实例变量：引用.实例变量名
  4. 通过=赋值将堆内存中实例变量的值进行修改

三、空指针异常

  • 该对象没有引用指向它了，Java中规定程序员无法直接操作堆内存，只能等待gc把垃圾对象回收走。

四、方法调用时参数的传递问题

  • Java中规定：参数传递的时候，和类型无关，不管你是基本数据类型，还是引用数据类型
